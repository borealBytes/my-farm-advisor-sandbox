## Writing Guidelines for This Template

> _Remove this section before distributing_

**The Pyramid Principle (Barbara Minto):**

1. Start with the answer/recommendation (top of pyramid)
2. Support with key arguments (second level)
3. Back arguments with data/evidence (base)

**C-Suite Writing Rules:**

- **Length:** 1–2 pages maximum for the main body. Executives read the first page; everything else is reference.
- **Sentences:** Short. Active voice. Subject-verb-object.
- **Numbers:** Always include units. Round to meaningful precision ($2.3M not $2,347,821).
- **Jargon:** Zero. If you must use an acronym, define it once.
- **Hedging:** Eliminate "it could be argued," "potentially," "it seems." State conclusions directly.
- **The "So what?" test:** After every sentence, ask "so what?" If you can't answer, cut the sentence.
- **The "Grandma test":** Could a smart non-expert understand this? If not, simplify.

**Common Mistakes:**

- ❌ Starting with background/history instead of the recommendation
- ❌ Burying the ask on page 3
- ❌ Using passive voice ("it was decided" → "we decided")
- ❌ Presenting options without a recommendation
- ❌ Including data without interpretation ("sales fell 12%" → "sales fell 12%, driven by X, requiring Y")
- ❌ Asking for "input" when you need a decision
