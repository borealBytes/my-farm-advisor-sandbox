# Mindmap — Intermediate (10–20 nodes)

Multi-category breakdown with 3–4 branches. Use for feature documentation and system overviews.

---

## Example: System Architecture

```mermaid
mindmap
 accTitle: Application Architecture Overview
 accDescr: Mindmap showing application architecture with frontend, backend, and infrastructure components

 root((🏗️ System))
  🎨 Frontend
   React app
   State management
   Component library
   Routing
  ⚡ Backend
   API Gateway
   Auth service
   Data service
   Queue workers
  💾 Data
   PostgreSQL
   Redis cache
   S3 storage
  🚀 DevOps
   Docker
   Kubernetes
   CI/CD pipeline
```

## Tips

- 3–4 first-level branches provides good coverage
- Go 3 levels deep when needed (root + 2 children)
- Balance branches (similar number of children)
- Use consistent emoji themes per category
- Consider color coding with emojis
