## Purpose

This policy establishes requirements for protecting company information assets.

---

## Scope

Applies to all employees, contractors, and third parties with access to company systems and data.

---

## Policy Statements

### 1. Access Control

- Access granted based on least privilege principle
- User accounts require approval
- Multi-factor authentication required
- Access reviewed quarterly

### 2. Password Management

| Requirement | Standard |
|-------------|----------|
| Minimum length | [X] characters |
| Complexity | Upper, lower, number, symbol |
| Expiration | [X] days |
| Reuse | Cannot reuse last [X] passwords |
| Lockout | After [X] failed attempts |

### 3. Data Classification

| Level | Description | Handling |
|-------|-------------|----------|
| Public | Can be shared freely | Standard |
| Internal | Company use only | Internal controls |
| Confidential | Restricted access | Encryption, approval |
| Restricted | Highly sensitive | Encryption, MFA, logging |

### 4. Acceptable Use

- Company systems for business purposes only
- No malicious software
- No unauthorized software
- Report security incidents immediately

### 5. Mobile Devices

- [ ] BYOD permitted
- [ ] Mobile device management required
- [ ] Encryption required
- Report lost/stolen devices within [X] hours

### 6. Remote Access

- VPN required for remote access
- Approved devices only
- Screen lock after [X] minutes
- No public Wi-Fi without VPN

### 7. Incident Response

Report incidents to: [security@company.com]

| Severity | Response Time |
|----------|---------------|
| Critical | 1 hour |
| High | 4 hours |
| Medium | 24 hours |
| Low | 72 hours |

### 8. Third-Party Access

- Background checks required
- Security agreements in place
- Access revoked upon termination
- Regular access reviews

## Compliance

Violations may result in disciplinary action, including termination.
